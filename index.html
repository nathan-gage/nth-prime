Technical note

Exact inversion of the prime-counting function

Abstract

Given a rank n, the program returns the unique prime pn. The computation is organized as a certification pipeline: estimate a coordinate, compute its exact prime-counting rank, correct the coordinate if the rank error is large, then perform a short exact segmented sieve to land on the target prime. No approximate step is trusted as a result.

1 Estimate

Use an asymptotic inverse to propose x0.

2 Count

Compute π(x0) exactly by Lehmer counting.

3 Correct

Move by local density and recount exactly.

4 Sieve

Segment-sieve the residual rank gap.

5 Certify

Return only the prime whose exact rank is n.

1. Definition and correctness invariant

The inverse is formulated as the first integer whose exact prime count reaches n. For n ≥ 1, correctness is certified by the two inequalities π(pn) = n and π(pn - 1) = n - 1. The program therefore reduces inversion to one exact global count and one exact local search.

2. Coordinate proposal by asymptotic inversion

The first coordinate x0 is a Cipolla-style asymptotic inverse. Let L = log n and ℓ = log L. Keeping terms through L-4 gives a coordinate whose rank error is small enough that the subsequent correction and segment search are bounded.

3. Exact rank certification by Lehmer counting

The rank r0 = π(x0) is computed exactly. The implementation uses a Lehmer decomposition with generated per-call prime tables. With a = π(x1/4), b = π(x1/2), and c = π(x1/3), the top-level count separates a partial sieve term from one-prime and two-prime subtraction terms.

4. Runtime-generated partial sieve data

The partial sieve function φ(x, a) counts integers not divisible by the first a primes. Small φ rows and direct rank data are generated during one invocation, used to answer exact subqueries, and discarded before the next measured call.

5. Rank-error correction

Let d0 = π(x0) - n. When the absolute error is material, the algorithm proposes a density correction using dπ/dx ≈ 1/log x. The corrected coordinate is counted exactly and accepted only if it reduces |d0|.

6. Local certified search

After the final exact count, only the residual rank error remains. If π(x) < n, the program sieves forward until it has observed n - π(x) primes; otherwise it sieves backward and skips π(x) - n primes. The segment is odd-only. Each base prime p contributes the first marked odd index and a stride of p odd slots.

Algorithm summary

  1. Compute x0 from the asymptotic inverse.
  2. Compute r0 = π(x0) exactly by Lehmer counting.
  3. If |r0 - n| is large, compute x1, recount exactly, and keep the coordinate with smaller rank error.
  4. Segment-sieve forward or backward from the certified coordinate until the nth prime is reached.

Complexity and honesty

Algorithmic analysis

Asymptotic shape

The estimator is constant-time with respect to generated tables. Exact counting dominates the observed cost near the 1B and 10B rank boundaries, while the final segmented sieve is proportional to the residual rank error times the local average gap, roughly O(Δ log x) odd candidates after correction.

Why exact counting dominates

Near 10B, the top-level Lehmer range contains tens of thousands of terms. Many small subqueries are answered directly from per-call generated rank data, but the residual recursive π(floor(x / pi)) calls remain the primary hot path.

Isolation boundary

The shipped WASM contains code, not an answer table. It does not include sampled anchors, rank-to-prime maps, or third-party prime libraries. Browser and CPU warmth can still affect wall-clock timings, but algorithmic state is reset across measured calls.
